# Настройка Базы Данных D1 и API Авторизации на Cloudflare

Этот каталог содержит исходный код для создания полноценной системы авторизации лаунчера **Cozy House** с использованием **Cloudflare D1** (SQL база данных) и **Cloudflare Workers**.

## Содержимое каталога
* `schema.sql` — схема базы данных D1 (таблицы пользователей и игровых сессий).
* `wrangler.toml` — конфигурационный файл Wrangler для развертывания проекта.
* `src/index.js` — код API-сервера Workers (регистрация, криптографическое хеширование PBKDF2, авторизация, валидация сессий и токенов).

---

## Пошаговая инструкция по развертыванию

### Шаг 1: Установка Wrangler
Установите CLI-инструмент Cloudflare Wrangler (если еще не установлен):
```bash
npm install -g wrangler
```
*Примечание: Если не хотите устанавливать глобально, используйте префикс `npx wrangler`.*

### Шаг 2: Авторизация в Cloudflare
Выполните вход в свою учетную запись Cloudflare:
```bash
wrangler login
```

### Шаг 3: Создание базы данных D1
Создайте новую базу данных SQL под названием `cozy-db`:
```bash
wrangler d1 create cozy-db
```
После создания D1 Wrangler выведет в консоль блок настроек с уникальным `database_id`.

### Шаг 4: Настройка wrangler.toml
Откройте файл `server/wrangler.toml` и замените `YOUR_D1_DATABASE_ID_HERE` на полученный `database_id`.
Пример:
```toml
database_id = "a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d"
```

### Шаг 5: Инициализация схемы таблиц (Создание структуры)
Примените SQL-схему из файла `schema.sql` к вашей созданной базе данных в облаке Cloudflare:
```bash
wrangler d1 execute cozy-db --remote --file=schema.sql
```
*(Для локального тестирования вместо `--remote` используйте флаг `--local`)*

### Шаг 6: Развертывание API
Опубликуйте API в облаке Cloudflare:
```bash
wrangler deploy
```
После завершения процесса Wrangler предоставит вам URL-адрес развернутого API (например, `https://cozy-house-auth.your-subdomain.workers.dev`).

---

## Интеграция с лаунчером (Клиентская часть)
После развертывания API вы сможете связать форму входа в лаунчере (`cozy-login.js`) с этим URL-адресом, выполняя стандартные Fetch-запросы:
* `POST /api/register` — для регистрации.
* `POST /api/login` — для входа в аккаунт.
